# 双车部署操作全流程（详细版）

> 适用场景：两台小车**目前没有任何程序**，从零开始部署双车联控。  
> 按 **阶段 0 → 1 → 2 → 3** 顺序做，不要跳步。

---

## 0. 一张图看懂谁干什么

```
  手机 App                主车（Car A）                 从车（Car B）
  ─────────              ──────────────                ──────────────
  填主车 IP               跑 ROS（占 :6000）              跑 slave_gateway.py
  连 :6000        ───>    ROS 照旧收指令、动车   ───>  监听 :6001 收主车转发
  开双车模式              + 执行后 relay 转发从车      执行同样指令
```

| 谁 | 要装什么 | 要放 config 吗 |
|----|----------|----------------|
| **手机** | 编译好的 App | **不需要** config 文件 |
| **主车** | `car/` 里一批 `.py` + **`config_master.json`** | **要，放在主车** |
| **从车** | `car/` 里一批 `.py` + **`config_slave.json`** | **要，放在从车** |

**结论：config 必须放在小车上**，和 Python 脚本在**同一目录**。  
手机 App 里填的 IP 是界面配置，**不能代替**小车上的 `config_*.json` 文件。

## 1. 配置文件说明（先看懂再动手）

仓库里配置文件**已经是正式文件名**，拷到小车上即可，**不用再 `cp` 改名**：

| 文件（`car/` 目录下） | 放哪台车 | 你要改什么 |
|----------------------|----------|------------|
| **`config_master.json`** | **主车** | 把 `slave_ip` 改成从车真实 IP |
| **`config_slave.json`** | **从车** | 一般不用改（默认监听 6001） |

### 1.1 主车上的 `config_master.json`（必建）

路径：**主车** `/home/jetson/smartcar/config_master.json`

```json
{
  "slave_ip": "192.168.1.12",
  "car_tcp_port": 6001,
  "dual_mode": true,
  "ack_timeout_ms": 300,
  "status_interval_sec": 1.0
}
```

| 字段 | 含义 | 你要改什么 |
|------|------|------------|
| `slave_ip` | 从车的 IP | **改成你从车的真实 IP** |
| `car_tcp_port` | 主车连从车的端口 | 一般保持 `6001` |
| `dual_mode` | 是否启用双车转发 | 保持 `true` |

### 1.2 从车上的 `config_slave.json`（必建）

路径：**从车** `/home/jetson/smartcar/config_slave.json`

```json
{
  "car_tcp_host": "0.0.0.0",
  "car_tcp_port": 6001
}
```

| 字段 | 含义 | 你要改什么 |
|------|------|------------|
| `car_tcp_host` | 监听地址 | 保持 `0.0.0.0`（表示允许外部连接） |
| `car_tcp_port` | 监听端口 | 保持 `6001`，与主车配置一致 |

### 1.3 手机 App 里的 IP 和 config 的关系

| 配置位置 | 内容 | 和 config 文件关系 |
|----------|------|-------------------|
| 手机 App | 主车 IP、端口 6000 | 手机**只连主车**，与 `config_master.json` 无关 |
| 手机 App | 双车模式 + 从车 IP | 连接成功后会发 `@CONFIG` 给主车，**可覆盖**主车里 `slave_ip` |
| 主车 `config_master.json` | 从车 IP | **主车程序启动时就读这个文件**；建议与 App 里填的一致 |
| 从车 `config_slave.json` | 监听 6001 | **只有从车程序读**，手机不读这个文件 |

**推荐做法**：主车 `config_master.json` 和 App 里填的从车 IP **写成同一个**，避免混乱。

## 2. 每台小车上要放哪些文件

统一目录：`/home/jetson/smartcar/`（路径可自定，但**脚本和 config 必须在同一文件夹**）。

### 2.1 从车（Car B）目录 — 最少 4 个文件

```
/home/jetson/smartcar/
├── slave_gateway.py          # 启动入口（监听 6001）
├── protocol.py               # 协议解析（被上面引用）
├── command_executor.py       # 执行 $...# 指令（接电机改这个）
└── config_slave.json         # ★ 已带好，拷过去后一般不用改
```

### 2.2 主车（Car A）目录 — 你已有 ROS 占 6000

主车 **不用改 App、不用改 ROS 怎么收 6000**，只在 ROS **执行完一条指令之后**，多转发一份给从车。

**不需要** `master_phone_server.py`（会和 ROS 抢 6000）。

```
/home/jetson/smartcar/
├── orchestrator.py
├── protocol.py
├── config_master.json            # ★ 改 slave_ip 为从车 IP
├── ros_bridge.py                 # 可选，一般不用动
└── relay_cmd.py                  # ★ 转发脚本，ROS 执行完指令后调它
```

| 文件 | 主车要不要 | 说明 |
|------|-----------|------|
| `relay_cmd.py` + `config_master.json` | **要** | 主车执行完指令后，用它转发从车 |
| `ros_bridge.py` | 不用管 | 给改 Python 源码用的，你可以忽略 |
| `command_executor.py` | **主车不需要** | ROS 里已有解析，别重复造轮子 |

**不用拷进 ROS 文件夹。** 放在 `/home/jetson/smartcar/` 即可；ROS 里加几行 `import`（见步骤 2.4）。

> 仅当主车**完全没有 ROS** 时，才用可选的 `master_phone_server.py` 单独监听 6000。

### 2.3 从 Desktop 放到 smartcar（你已上传到小车）

文件在 **`~/Desktop/` 桌面上**（没有 `car` 子文件夹）时，在**对应小车终端**执行：

先看一下桌面上有哪些文件：

```bash
ls ~/Desktop
```

#### 从车：Desktop → `/home/jetson/smartcar`

```bash
mkdir -p /home/jetson/smartcar

cp ~/Desktop/slave_gateway.py \
   ~/Desktop/protocol.py \
   ~/Desktop/command_executor.py \
   ~/Desktop/config_slave.json \
   /home/jetson/smartcar/

ls /home/jetson/smartcar
```

应看到 4 个文件：`slave_gateway.py`、`protocol.py`、`command_executor.py`、`config_slave.json`。

#### 主车：Desktop → `/home/jetson/smartcar`

```bash
mkdir -p /home/jetson/smartcar

cp ~/Desktop/orchestrator.py \
   ~/Desktop/protocol.py \
   ~/Desktop/config_master.json \
   ~/Desktop/ros_bridge.py \
   ~/Desktop/relay_cmd.py \
   /home/jetson/smartcar/

ls /home/jetson/smartcar
```

应看到 5 个文件：`orchestrator.py`、`protocol.py`、`config_master.json`、`ros_bridge.py`、`relay_cmd.py`。

> 之后所有 `python3`、`cd`、ROS 里的 `sys.path` 都指向 **`/home/jetson/smartcar`**，不要在 Desktop 下运行。

#### 若你上传的是整个 `car` 文件夹

只有 `~/Desktop/car/` 存在时，把上面路径里的 `~/Desktop/xxx` 改成 `~/Desktop/car/xxx` 即可。

## 3. 阶段 0：三台设备连同一网络

**在哪做**：主车、从车、手机

| 步骤 | 操作 | 怎么算成功 |
|------|------|------------|
| 0.1 | 主车、从车上电，连同一个 WiFi | 两台车都能 `ping` 通路由器 |
| 0.2 | 主车终端执行 `ip addr`，记下 IP，例如 `192.168.1.11` | 写下来：**主车IP = ______** |
| 0.3 | 从车终端执行 `ip addr`，记下 IP，例如 `192.168.1.12` | 写下来：**从车IP = ______** |
| 0.4 | 手机连同一 WiFi | 手机能打开 App |

## 4. 阶段 1：配置并启动从车（先做从车）

**在哪做**：只在**从车**上操作

### 步骤 1.1 登录从车并确认文件已就位

若还没从 Desktop 拷到 smartcar，先做 [2.3 节](#23-从-desktop-放到-smartcar你已上传到小车)。

```bash
ssh jetson@从车IP
cd /home/jetson/smartcar
```

### 步骤 1.2 确认文件都在

```bash
ls -la
```

必须能看到：

- `slave_gateway.py`
- `protocol.py`
- `command_executor.py`
- `config_slave.json`

### 步骤 1.3 检查从车 config（一般不用改）

```bash
cat config_slave.json
```

能看到上面 JSON 即成功。

### 步骤 1.4 启动从车程序

```bash
cd /home/jetson/smartcar
python3 slave_gateway.py
```

**成功标志**（终端里出现）：

```
slave car-tcp listening on 0.0.0.0:6001
```

此时**不要关这个终端**。若关窗口程序就停了。

后台运行（可选）：

```bash
cd /home/jetson/smartcar
nohup python3 slave_gateway.py > slave.log 2>&1 &
```

### 步骤 1.5 自检（在电脑或主车上）

```bash
nc -zv 从车IP 6001
```

出现 `open` 或 `succeeded` 表示从车就绪。

**阶段 1 完成标志**：从车 6001 在监听，且 `config_slave.json` 在 `/home/jetson/smartcar`。

---

## 5. 阶段 2：配置并启动主车

**在哪做**：只在**主车**上操作

### 步骤 2.1 登录主车并确认文件已就位

若还没从 Desktop 拷到 smartcar，先做 [2.3 节](#23-从-desktop-放到-smartcar你已上传到小车)。

```bash
ssh jetson@主车IP
cd /home/jetson/smartcar
```

### 步骤 2.2 确认文件都在

```bash
ls -la
```

必须能看到：

- `orchestrator.py`
- `protocol.py`
- `config_master.json`
- `relay_cmd.py`

### 步骤 2.3 修改主车 config（★ 必做：改从车 IP）

```bash
vi config_master.json
```

**务必修改** `slave_ip` 为你的从车真实 IP，例如：

```json
{
  "slave_ip": "192.168.1.12",
  "car_tcp_port": 6001,
  "dual_mode": true,
  "ack_timeout_ms": 300,
  "status_interval_sec": 1.0
}
```

验证：

```bash
cat config_master.json
```

确认 `slave_ip` 不是示例里的错 IP。

### 步骤 2.4 主车 ROS 加「转发」—— 你已有解析，只加几行

这一步需要改ros里的代码，我们暂时采用的是异步执行指令发送

### 步骤 2.5 自检

```bash
nc -zv 主车IP 6000
```

**阶段 2 完成标志**：ROS 占 6000，执行指令后会 relay 到从车，`config_master.json` 里从车 IP 正确。

## 6. 阶段 3：手机 App 连接

**在哪做**：只在**手机**上操作（不需要往手机拷 config 文件）

| 步骤 | App 里填什么 | 填什么值（示例） |
|------|--------------|------------------|
| 3.1 | 主车 IP | `192.168.1.11`（你的主车IP） |
| 3.2 | TCP 端口 | `6000` |
| 3.3 | Video 端口 | `6500`（暂无视频可先填） |
| 3.4 | 打开「双车模式」 | 开 |
| 3.5 | 从车 IP | `192.168.1.12`（与主车 config 里一致） |
| 3.6 | 从车端口 | `6001` |
| 3.7 | 点「连接」 | Toast 提示连接成功 |

进入「单独控制」页面，看顶部状态条：

| 显示文字 | 含义 |
|----------|------|
| 双车同步中 | 正常，两台车都在跟指令 |
| 从车离线 · 主车单控 | 从车没起来或 IP 错了，主车还能动 |
